# 美容连锁门店经营分析看板系统 - 系统部署说明

## 1. 系统概述

美容连锁门店经营分析看板系统是一套面向美容连锁企业的数据分析与决策支持平台。系统采用前后端分离架构，前端使用 Vue 3 + Ant Design Vue + Plotly 构建，后端使用 Python Flask + Pandas 提供 API 服务。

## 4. 详细部署指南

### 4.1 后端部署

#### 4.1.1 安装 Python 依赖

```bash
cd backend
pip install -r requirements.txt
```

#### 4.1.2 生成模拟数据

```bash
python utils/generate_data.py
```

#### 4.1.3 开发模式运行

```bash
python app.py
```

#### 4.1.4 生产模式部署

推荐使用 Gunicorn + Nginx 部署：

**安装 Gunicorn**:
```bash
pip install gunicorn
```

**启动 Gunicorn**:
```bash
gunicorn -w 4 -b 0.0.0.0:5000 app:create_app()
```

**使用 systemd 管理服务**:

创建 `/etc/systemd/system/beauty-dashboard.service`:
```ini
[Unit]
Description=Beauty Dashboard Backend
After=network.target

[Service]
Type=simple
User=www-data
WorkingDirectory=/path/to/backend
ExecStart=/usr/bin/gunicorn -w 4 -b 127.0.0.1:5000 app:create_app()
Restart=always
RestartSec=5

[Install]
WantedBy=multi-user.target
```

启动服务:
```bash
sudo systemctl start beauty-dashboard
sudo systemctl enable beauty-dashboard
```

### 4.2 前端部署

#### 4.2.1 安装依赖

```bash
npm install
```

#### 4.2.2 开发模式

```bash
npm run dev
```

#### 4.2.3 生产构建

```bash
npm run build
```

构建产物将输出到 `dist` 目录。

#### 4.2.4 Nginx 部署

**安装 Nginx**:
```bash
sudo apt install nginx
```

**配置 Nginx**:

创建 `/etc/nginx/sites-available/beauty-dashboard`:
```nginx
server {
    listen 80;
    server_name your-domain.com;

    # 前端静态文件
    location / {
        root /path/to/dist;
        index index.html;
        try_files $uri $uri/ /index.html;
    }

    # API 代理
    location /api/ {
        proxy_pass http://127.0.0.1:5000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }

    # Gzip 压缩
    gzip on;
    gzip_types text/plain text/css application/json application/javascript;
    gzip_min_length 1024;
}
```

启用站点:
```bash
sudo ln -s /etc/nginx/sites-available/beauty-dashboard /etc/nginx/sites-enabled/
sudo nginx -t
sudo systemctl reload nginx
```

## 5. Docker 部署

### 5.1 后端 Dockerfile

创建 `backend/Dockerfile`:
```dockerfile
FROM python:3.11-slim

WORKDIR /app

COPY requirements.txt .
RUN pip install --no-cache-dir -r requirements.txt

COPY . .

EXPOSE 5000

CMD ["python", "app.py"]
```

### 5.2 前端 Dockerfile

创建 `Dockerfile`:
```dockerfile
FROM node:18-alpine as builder

WORKDIR /app

COPY package*.json ./
RUN npm ci

COPY . .
RUN npm run build

FROM nginx:alpine

COPY --from=builder /app/dist /usr/share/nginx/html
COPY nginx.conf /etc/nginx/conf.d/default.conf

EXPOSE 80

CMD ["nginx", "-g", "daemon off;"]
```

创建 `nginx.conf`:
```nginx
server {
    listen 80;
    server_name localhost;

    root /usr/share/nginx/html;
    index index.html;

    location / {
        try_files $uri $uri/ /index.html;
    }

    location /api/ {
        proxy_pass http://backend:5000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }
}
```

### 5.3 Docker Compose

创建 `docker-compose.yml`:
```yaml
version: '3.8'

services:
  backend:
    build: ./backend
    ports:
      - "5000:5000"
    volumes:
      - ./backend/data:/app/data
    restart: unless-stopped

  frontend:
    build: .
    ports:
      - "80:80"
    depends_on:
      - backend
    restart: unless-stopped
```

启动服务:
```bash
docker-compose up -d
```

## 6. 配置说明

### 6.1 前端配置

#### 6.1.1 开发环境代理配置

在 `vite.config.ts` 中配置代理：

```typescript
export default defineConfig({
  server: {
    proxy: {
      '/api': {
        target: 'http://127.0.0.1:5000',
        changeOrigin: true,
      },
    },
  },
})
```

#### 6.1.2 生产环境 API 地址

在生产环境中，建议通过 Nginx 反向代理将 `/api` 请求转发到后端服务，无需修改前端代码。

### 6.2 后端配置

#### 6.2.1 端口配置

在 `app.py` 中修改端口：

```python
if __name__ == '__main__':
    app.run(debug=True, host='0.0.0.0', port=5000)
```

#### 6.2.2 数据目录配置

CSV 数据文件默认存放在 `backend/data/` 目录下，可通过修改 `utils/csv_loader.py` 中的 `DATA_DIR` 变量更改路径。

#### 6.2.3 CORS 配置

默认已开启全域名 CORS 支持，如需限制域名，修改 `app.py` 中的 CORS 配置：

```python
CORS(app, resources={r"/api/*": {"origins": ["https://your-domain.com"]}})
```

## 9. 常见问题

### 9.1 前端无法连接后端

**问题**: 前端请求 API 时返回 HTML 或报错

**解决方案**:
1. 确认后端服务已启动
2. 检查 API 地址配置是否正确
3. 开发环境检查 `vite.config.ts` 中的代理配置
4. 生产环境检查 Nginx 反向代理配置

### 9.2 端口被占用

**问题**: 启动时提示端口已被占用

**解决方案**:
- 修改 `app.py` 中的后端端口
- 修改 `vite.config.ts` 中的前端端口
- 或使用 `lsof -i :5000` 查找并关闭占用进程

### 9.3 CSV 数据文件缺失

**问题**: 启动后端时报错找不到 CSV 文件

**解决方案**: 运行 `python utils/generate_data.py` 生成模拟数据

### 9.4 跨域问题

**问题**: 浏览器报 CORS 错误

**解决方案**:
- 开发环境使用 Vite 代理
- 生产环境使用 Nginx 反向代理
- 或确保后端已启用 CORS（默认已启用）

## 10. 性能优化建议

### 10.1 前端优化

- 使用代码分割，按需加载页面
- 启用 Gzip/Brotli 压缩
- 使用 CDN 加速静态资源
- 优化图表渲染，大数据量时使用抽样

### 10.2 后端优化

- 启用 CSV 数据缓存（已实现）
- 使用 Gunicorn 多进程部署
- 考虑迁移到数据库存储
- 添加数据接口缓存（如 Redis）

### 10.3 数据库优化（迁移后）

- 建立合适的索引
- 使用分区表处理大数据量
- 定期清理历史数据
- 读写分离
